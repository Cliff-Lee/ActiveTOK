# Model Failure Challenge Worksheet

Name: ________________________________  Class: __________________  Date: __________________

## Learning Focus

Models simplify reality productively.

## Student Prompt

Inspect Model Failure Challenge Stimulus A. Record one first claim, the evidence you used, your confidence, and what would make you revise.

## Concrete Stimulus Packet

### Model Failure Challenge Stimulus A

Student-facing stimulus 1: A simple weather, disease, or traffic model that works until one assumption changes. Identify the observation, variable, control, anomaly, or missing method detail before making a claim.

Teacher cue: Students inspect this exact card first and commit to a claim before hearing anyone else.

### Model Failure Challenge Stimulus B

Student-facing stimulus 2: An anomaly card that forces students to revise a model. Identify the observation, variable, control, anomaly, or missing method detail before making a claim.

Teacher cue: Use this for comparison, a second round, or a partner challenge.

### Model Failure Challenge Reveal / Twist

Student-facing stimulus 3: A model limits table: useful for, weak when, assumptions, and missing variables. Identify the observation, variable, control, anomaly, or missing method detail before making a claim.

Teacher cue: Teacher reveal: connect the changed response to the big idea — Models simplify reality productively.


## Actual Lesson Questions

1. After inspecting Model Failure Challenge Stimulus A, what is your first knowledge claim?
   - Teacher key/cue: Teacher cue: look for a clear claim, not just a description.
2. Which exact detail from Model Failure Challenge Stimulus A did you use as evidence?
   - Teacher key/cue: Teacher cue: students should name visible words, data, source features, method features, or contextual clues.
3. What did you infer beyond the evidence?
   - Teacher key/cue: Teacher cue: this is where assumptions, framing, models, or perspective usually appear.
4. How confident are you: 50%, 60%, 70%, 80%, 90%, or 100%?
   - Teacher key/cue: Teacher cue: confidence is data for the debrief, not a grade.
5. What missing information would most improve the knowledge claim?
   - Teacher key/cue: Teacher cue: push for method, source, context, comparison, or definitions.
6. Are models useful because they are false?
   - Teacher key/cue: Teacher cue: students should move from the classroom example to a general TOK issue.

## Activity Steps

1. Groups identify variables missing from the model: prior knowledge, sleep, stress, revision quality, teaching, test difficulty.
2. They draw a revised model with extra variables.
3. They decide whether the original model is useless, useful or dangerous.
4. Discuss when a model should be rejected.

## Teacher Language

- Write your first judgement before we discuss it; the first answer is useful evidence.
- Separate what you noticed from what you inferred.
- What would make this knowledge claim more reliable?
